# WeRead to Notion

Chrome Manifest V3 插件，用于把微信读书书架和阅读进度同步到 Notion 数据库。

## 功能

- 读取当前 Chrome 已登录的微信读书网页版会话。
- 在同一个扩展页内用顶部页签区分书架同步、划线同步和配置页。
- 书架同步页读取书架、默认全选、手动同步，并配置书架字段条目。
- 划线同步页读取有笔记的书，展示每本书的划线/想法数量，并支持按单本书同步。
- 划线同步页可独立配置划线字段条目。
- 配置页设置 Notion 内部集成密钥、数据库 URL/ID。
- 书名默认写入 Notion 数据库自带的 title 字段。
- 字段条目可选择 Notion 字段、微信读书字段或自定义内容。
- 可将微信读书封面设置为 Notion 页面封面。
- 使用 `WeRead ID` 字段查询已有页面，重复同步时更新而不是重复创建。
- 字段可独立设置是否覆盖更新。
- 划线同步采用“一本书一个 Notion 页面”，原文和想法写入页面内容；重复同步会替换该页内容，避免重复追加。

## Notion 数据库要求

目标数据库必须包含：

- Notion title 字段：字段名不限，扩展会自动识别并写入书名。
- `WeRead ID`：建议使用 rich text 或 select 字段，用于去重。

可选字段类型：

- 封面：url 或 files。
- 阅读进度：number。
- 作者、WeRead ID：rich text 或 select。
- URL：url 或 rich text。
- 阅读状态：select、status 或 rich text。

如果使用 status 字段，请先在 Notion 中准备好“未开始”“阅读中”“已读完”三个选项。

划线同步数据库必须包含：

- Notion title 字段：字段名不限，扩展会自动识别。
- 建议添加 `WeRead ID` 或 `Book ID`：rich text 或 select 字段，用于更稳的去重。没有该字段时，扩展会按固定标题 `《书名》划线与想法` 查找已有页面。

划线同步数据库可按需添加字段条目：

- 可同步作者、微信读书链接、Book ID、笔记总数、划线数量、想法数量、最后同步时间。
- 也可以对 status、select、rich text 等字段写入自定义内容。

## 开发

```bash
npm install
npm run build
```

构建后在 Chrome 打开 `chrome://extensions`，启用开发者模式，选择 `dist` 目录加载已解压的扩展。

## 使用

1. 在 Chrome 中登录微信读书网页版。
2. 创建 Notion 内部集成，复制密钥，并把目标数据库授权给该集成。
3. 打开扩展设置页，填写 Notion 密钥和数据库 URL/ID，点击“验证数据库”。
4. 如需同步划线，填写划线数据库 URL/ID，点击“验证划线数据库”。
5. 到“书架同步”和“划线同步”分别配置字段条目并保存。
6. 点击扩展图标打开书架同步，点击“读取书架”，勾选要同步的书，点击“同步到 Notion”。
7. 切到“划线同步”，点击“读取划线书籍”，选择一本书后点击“立即同步”。

## 说明

微信读书网页接口不是公开稳定 API。本项目把微信读书读取逻辑隔离在 `src/services/weread.ts`，后续若接口变化，只需优先调整这个适配层。
